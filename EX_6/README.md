# EX_7 — Data Governance in the Cloud

In this 60-minute group exercise, you apply core **data governance** concepts to a practical cloud-based scenario. You will identify important data, define basic governance rules, assign responsibilities, assess maturity, and research one or two market solutions that could support data governance in the cloud.

You will:
- Choose one cloud-based scenario  
- Identify 3–5 important data types  
- Define a mini data governance policy  
- Assign data governance roles within the group  
- Research one or two data governance tools or platforms  
- Assess the organization’s data governance maturity level  
- Identify risks of poor data governance  
- Propose three concrete recommendations  
- Prepare a one-slide or one-page summary and present it briefly  

> Full instructions are in `EX_6_instructions.docx`

---

## Exercise Setup

This is a **60-minute group exercise**.

- Group size: **3 students**
- Final deliverable: **one slide or one-page summary**
- Presentation: **2 minutes per group**

Each student should take one role:

- **Data Owner** — decides which data is important and who may use it  
- **Data Steward** — ensures data quality, documentation, and correct use  
- **Cloud/Data Security Officer** — proposes security, access control, and cloud protection measures  

---

## Cloud Scenarios

Choose one of the following scenarios:

- **Online store**  
  Customer data, orders, payments, product views, and website behavior.

- **Hospital or private clinic**  
  Patient records, appointments, lab results, and treatment data stored in the cloud.

- **University**  
  Student records, grades, attendance, online courses, and learning analytics.

- **Bank or fintech application**  
  Customer accounts, transactions, loan applications, and fraud detection data.

- **Smart city**  
  Data from sensors, traffic systems, public transport, cameras, and environmental monitoring systems.

---

## What Your Group Needs to Prepare

### 1. Key Data

Identify **3–5 important types of data** used in your scenario. For each data type, briefly explain why it is important.

Example:

| Data type | Why it is important |
|---|---|
| Customer data | Used for user accounts, support, and personalization |
| Payment data | Important for transactions, security, and reporting |
| Website behavior data | Used for analytics and recommendations |

---

### 2. Mini Data Governance Policy

Create a simple data governance policy for your scenario. For each area, write **one concrete rule or action**.

| Governance area | Guiding question |
|---|---|
| Data quality | How will you make sure the data is accurate and consistent? |
| Data availability | Who should have access to the data and when? |
| Data usability | How should the data be structured, documented, or labeled? |
| Data integrity | How will you prevent data loss or unauthorized changes? |
| Data security | Which data is sensitive and how should it be protected? |

---

### 3. Roles and Responsibilities

Briefly explain what each role would do in your selected scenario.

| Student | Role | Main responsibility |
|---|---|---|
| Student 1 | Data Owner | Decides which data is important and who may use it |
| Student 2 | Data Steward | Ensures data quality, documentation, and correct use |
| Student 3 | Cloud/Data Security Officer | Proposes security, access control, and cloud protection measures |

---

### 4. Market Solution Research

Research **one or two existing data governance tools or platforms** that could help your organization.

Examples:

- Microsoft Purview  
- Google Cloud Dataplex  
- AWS Lake Formation  
- Collibra  
- Informatica Intelligent Data Management Cloud  
- Alation  
- Atlan  
- Apache Atlas  
- OpenMetadata  
- DataHub  

For each selected tool, answer:

- What is the tool used for?
- Which governance problems does it help solve?
- Why is it suitable for your selected scenario?
- Is it more suitable for a small, medium, or large organization?

A detailed technical comparison is not required. Focus on the connection between the tool and your governance needs.

---

### 5. Data Governance Maturity Level

Assess the organization’s current data governance maturity level. Choose one level:

| Level | Description |
|---|---|
| Level 1: Aware | The organization knows it needs data governance, but has no clear rules |
| Level 2: Reactive | Some rules exist, but they are incomplete or inconsistently applied |
| Level 3: Proactive | The organization actively introduces rules, roles, and controls |
| Level 4: Managed | The organization has formal policies, monitoring, standards, and tools |

Explain your choice in **2–3 sentences**.

---

### 6. Risks of Poor Data Governance

Identify one example for each type of impact:

| Impact type | Example |
|---|---|
| Financial impact | Fines, wrong reports, revenue loss, high correction costs |
| Reputational impact | Loss of trust from customers, patients, students, or citizens |
| Process impact | Bad decisions because of inaccurate, missing, or unavailable data |

---

### 7. Recommendations

Propose **three concrete recommendations** for improving data governance in your scenario.

Examples:

- Define clear access rules for sensitive data  
- Introduce a data catalog to make datasets easier to find and understand  
- Assign data owners and data stewards for important datasets  
- Use automated data quality checks  
- Classify sensitive data and apply encryption  
- Regularly audit data access and usage  

---

## Final Output

At the end of the exercise, each group should submit or present **one slide or one-page summary** containing:

1. Selected scenario  
2. Key data types  
3. Mini data governance policy  
4. Roles and responsibilities  
5. Selected market solution(s)  
6. Maturity level  
7. Main risks  
8. Three recommendations

## Presentation

Each group will briefly present its results in **2 minutes**.

The presentation should answer:

- What scenario did you choose?
- What data is most important?
- What are your main governance rules?
- Which tool or platform would you use and why?
- What is the current maturity level?
- What are your three main recommendations?

---

## Expected Learning Outcomes

By the end of this exercise, you should be able to:

- Identify key governance needs in a cloud-based data scenario  
- Define a basic data governance policy  
- Explain the roles of Data Owner, Data Steward, and Cloud/Data Security Officer  
- Evaluate the maturity level of an organization’s data governance approach  
- Connect governance problems with existing market solutions  
- Propose practical recommendations for improving data governance in the cloud
